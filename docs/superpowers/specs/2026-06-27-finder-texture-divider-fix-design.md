# Finder 纹理与分割线修复设计

## 目标

仅修复 Finder 窗口的两个视觉问题：左侧栏与主内容区分割线旁的浅色竖条，以及不可见的纸张纹理。现有排版、颜色、内容和交互保持不变。

## 参考实现

线上参考模块使用一个独立的绝对定位纹理层覆盖 Finder 窗口。纹理层位于窗口底色之上、内容之下，使用 `paper-texture.jpg`、`500px` 平铺、`mix-blend-mode: multiply` 和 `opacity: 0.3`。左侧栏拥有半透明米色背景及单独的右边框，右侧主内容区保持透明。

## 修复方案

1. 在 `.finder-window` 内增加独立的 `.finder-paper-texture` 元素，替代当前的 `::before` 纹理伪元素。
2. 纹理层绝对铺满窗口，设为 `z-index: 1`、不可响应指针，并保持 `500px` 平铺、`multiply`、透明度 `0.3`。
3. 标题栏和 Finder 主体统一置于 `z-index: 2`。
4. 保留 `.finder-sidebar` 的半透明背景和唯一的 1px 右边框。
5. 将 `.finder-viewport` 与 `.finder-grid` 的额外白色背景改为透明，消除边界处的叠色竖条，并让底层纹理连续可见。
6. 保持 `.finder-window` 和 `.finder-body` 的裁切规则，避免纹理或内部内容越过圆角边界。

## 验收标准

- 左侧栏右侧仅出现一条连续的 1px 分割线，没有额外浅色长条。
- 纸张纹理连续覆盖标题栏、侧栏和主内容区，参数为 `500px repeat`、`multiply`、`opacity: 0.3`。
- Finder 的尺寸、圆角、阴影、文件夹网格、详情面板和切换交互不发生变化。
- 桌面与移动端布局均无横向溢出或纹理越界。

## 验证

启动本地静态站点，在桌面宽度检查默认项目视图及打开详情后的分栏边界；切换 Snapshot、Achievements 和 Garden，确认纹理连续；再在移动端宽度检查圆角裁切与横向溢出。
