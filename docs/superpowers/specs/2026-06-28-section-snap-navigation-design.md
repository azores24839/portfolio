# 分段式滚动交互设计

## 目标

将作品集首页组织为具有 PPT 节奏的四个顶层滚动段落：Hero、Finder 项目区、作品墙、游乐场。联系方式与页脚并入游乐场末屏。轻微滚轮输入不应误触翻页；有效翻页需要平滑、单屏、稳定对齐，同时不破坏 Finder 详情等内部滚动区域。

## 方案

采用 CSS 原生滚动吸附为基础、桌面端 JavaScript 阈值控制为增强的混合方案。

- `html` 作为唯一的页面滚动容器，使用纵向强制吸附。
- 四个顶层 section 使用 `scroll-snap-align: start` 和至少一视口高度。
- 移动端保留浏览器原生触摸与惯性滚动，由 CSS 在手势结束后完成对齐。
- 精确指针设备上累计滚轮输入，达到阈值后只移动一屏，并在动画期间短暂锁定重复输入。
- 导航锚点仍使用原生 hash 定位和已有平滑滚动。
- 用户开启减少动态效果时，关闭强制吸附、阈值拦截和平滑动画。

## DOM 与布局

联系方式、复制提示和页脚移动到 `.playground` 内，成为最后一屏的底部内容。顶层吸附目标只选择 `main > section`，不会误选 Finder 内部的嵌套 section。

桌面端每个顶层 section 使用动态视口单位 `100svh`，并保留现有内容的最低高度约束。内容真实高度超过视口时不裁切，而是允许 section 变高；吸附点仍位于 section 顶部。

Hero 在桌面端形成独立、裁切的舞台，避免绝对定位物件进入 Finder 屏。Finder 屏内部保留 64px 顶部与 40px 底部呼吸区；这段留白属于 Finder section，不形成可单独停留的滚动区域。Finder 窗口高度从剩余视口空间计算，section 总高度仍为一屏。

桌面增强使用 `<main>` 作为唯一物理运动层。达到阈值后，页面逻辑位置立即对齐目标 section，同时通过反向父容器位移保持视觉连续；随后用逐帧弹簧积分将父容器偏移收敛到零。输入速度作为弹簧初速度，参数为 `mass: 1`、`stiffness: 190`、`damping: 25`，速度限制为 `1600px/s`。因此快速和缓慢手势具有不同惯性，目标附近会自然产生约 1–3px 的轻微 overshoot。所有子元素只随 `<main>` 整体运动，不分别添加位移、缩放、stagger 或 bounce。

物理过程期间临时关闭 CSS scroll snap 与平滑滚动，达到距离小于 `0.35px` 且速度小于 `6px/s` 的平衡条件后，清除父容器 transform、精确校正滚动位置并恢复原生吸附。新的桌面滚轮或导航输入可以重新定向弹簧，并继承当前速度。

桌面滚轮使用手势锁存防止触控板惯性尾部连续翻页。累计输入达到 `80px` 后，一次连续手势最多选择一个相邻 section；目标会保持到弹簧 settle 且滚轮静默 `180ms`。间隔超过 `180ms` 的输入会被识别为新手势，即使上一段弹簧仍在收敛也可以重新定向。同方向惯性事件不会继续推进目标，反方向重新累计到阈值时可以返回本次手势的起点。

## 滚轮阈值控制

桌面增强仅在同时满足以下条件时启用：精确指针、非减少动态模式、视口宽度大于移动端断点。

- 将归一化后的 `deltaY` 累计到同方向缓冲区。
- 方向变化或输入停顿后清空累计量。
- 达到约 100px 阈值时，根据当前最接近视口的 section 移动到相邻 section。
- 使用 `scrollIntoView({ behavior: "smooth", block: "start" })`。
- 约 700ms 内不接受第二次翻页，避免触控板惯性跳过多屏。
- 水平手势占主导、组合键滚动和表单控件交互不被接管。

## 内部滚动保护

事件目标向上查找可纵向滚动的祖先。祖先沿滚动方向仍有剩余距离时，页面控制器不调用 `preventDefault()`，让 Finder 详情、列表和其他模块正常滚动。只有内部容器到达对应边界后，继续滚动才参与页面翻屏阈值。

## 键盘、触摸与无障碍

- 不拦截触摸事件，移动端保持系统手势。
- 不覆盖浏览器原生 Tab、空格、PageUp、PageDown、Home 和 End 行为；CSS 负责最终吸附。
- 减少动态模式中使用 `scroll-behavior: auto` 和 `scroll-snap-type: none`。
- section 不新增无意义的焦点顺序。

## 验证

- 桌面滚轮小幅输入不会换屏，越过阈值后恰好移动一屏。
- 触控板惯性不会连续跳过多个 section。
- Finder 详情能完整内部滚动；抵达边界后才切换页面。
- 导航链接仍准确定位到关于、作品和游乐场。
- 手机尺寸下可以原生滑动并在停止后吸附。
- 所有 section 停止时顶部对齐，无横向溢出；减少动态模式不强制吸附。
- Hero 内容不会出现在 Finder 屏；Finder 顶部和底部留白稳定存在。
- 物理过程中只有 `<main>` 具有 transform，所有子元素保持相对静止。
- 不同输入速度会改变惯性，但最终精确落在目标 section；落点只出现克制 overshoot。
